## 벽돌깨기 - 게임 분석 및 개발 가이드

본 문서는 "벽돌깨기" 게임의 핵심 시스템, 규칙, 스킬/아이템 상세 설명 및 향후 개발/수정 시 참고할 수 있는 가이드를 제공합니다.

---

### 1. 개발 및 수정 가이드

`game.js` 파일은 게임 실행 시 초기 로드되는 기본 파일 입니다. `Game` 전역 객체를 통해 모든 변수와 함수에 접근하며, 각 시스템 별 별도의 `js` 파일을 참조합니다.
`player.js` 파일은 게임 내 플레이어와 관련된 기본적인 로직과 UI, 관련 설정을 담당합니다.
`leveling.js` 파일은 게임 내 플레이어의 레벨업과 관련된 로직 UI, 관련 설정을 담당합니다.
`enemies.js` 파일은 게임 내 적과 관련된 기본적인 로직과 UI, 관련 설정을 담당합니다.
`balls.js` 파일은 게임 내 공과 관련된 기본적인 로직과 UI, 관련 설정을 담당합니다.
`skills.js` 파일은 게임 내 스킬과 관련된 로직과 관련 설정을 담당합니다.
`items.js` 파일은 게임 내 아이템과 관련된 로직과 관련 설정을 담당합니다.

> 모든 `js` 파일 내 핵심 설정은 `전역변수` 로 지정하여 상단에 정의하여 사용합니다.
> 모든 `js` 파일 내 핵심 로직은 `전역변수` 를 최대한 활용하여 코드의 중복을 지양합니다.

#### 1.1. 전역 `Game` 객체 활용

모든 게임 상태 변수, 설정, 핵심 함수들은 `Game` 전역 객체의 속성으로 관리됩니다.
예: `Game.ballDamage`, `Game.characterHP`, `Game.updateHUD()`.

#### 1.2. 게임 루프 (`Game.loop`, `Game.update`, `Game.draw`)

*   **`Game.loop()`**: `requestAnimationFrame`을 사용하여 게임의 메인 루프를 실행합니다. `Game.update()`와 `Game.draw()`를 호출합니다.
*   **`Game.update()`**: 게임의 모든 물리, 상태 변화, 충돌 판정, 타이머 업데이트 등 논리적인 부분을 처리합니다.
*   **`Game.draw()`**: `Game.update()`에서 계산된 상태를 바탕으로 화면에 모든 요소를 그립니다.

#### 1.3. 주요 데이터 구조

*   **`Game.ENEMY_TYPES`**: 적의 종류, 기본 HP, 출현 가중치, 색상, 등급별 타격음 파일 경로를 정의합니다.
    *   `type`: 적의 종류 (NORMAL, ELITE, UNIQUE, SPECIAL, BOSS)
    *   `hp`: 기본 HP
    *   `weight`: 출현 가중치 (높을수록 자주 나옴, 0~1 사이의 값)
    *   `color`: 기본 색상 배열 (이미지 로드 실패 시 배열 내 `RGB 코드` 랜덤 사용)
    *   `hitSoundSrc`: 타격 효과음 파일 경로 (음원 로드 실패 시 타격 효과음 미사용)
*   **`Game.UPGRADES_DATA`**: 레벨업 시 선택 가능한 스킬 및 아이템의 정보를 정의합니다.
    *   `id`: 고유 ID
    *   `name`: 표시 이름
    *   `type`: 'skill' 또는 'item'
    *   `maxLevel`: 최대 레벨 (스킬: default 99, 아이템: default 3)
    *   `description(level)`: 현재 레벨에 따른 설명을 반환하는 함수
    *   `apply()`: 해당 업그레이드를 적용하는 함수
    *   `currentLevel()`: 현재 업그레이드 레벨을 반환하는 함수

#### 1.4. 이미지 및 사운드 파일

*   **이미지**: `images/` 디렉토리 내부의 `*.png` 파일을 사용합니다. `Game.spriteMap`에 정의된 스프라이트 시트 레이아웃에 따라 적 이미지를 렌더링합니다.
    *   실제 이미지와 화면 렌더링 사이즈가 다른 경우, 원본 이미지의 비율 대로 `resize, crop` 후 사용합니다.
*   **사운드**: `sounds/` 디렉토리 내부의 `*.mp3, *.wav` 파일을 사용합니다. `Game.playSound()` 함수를 통해 재생됩니다.

#### 1.5. 난이도 및 밸런스 조정

*   **`Game.ENEMY_TYPES`**: 적의 `hp`, `weight`를 조정하여 기본 난이도와 적 출현 빈도를 조절할 수 있습니다.
*   **`Game.ENEMY_HP_BONUS`**: 5턴마다 증가하는 적 HP 보너스 값을 `(Game.turns / 5) * 50` 공식에서 `50` 부분을 조정하여 난이도 상승 곡선을 조절할 수 있습니다.
*   **`Game.BALL_DAMEGE`, `Game.BALL_COUNT`**: 플레이어의 기본 공격력과 공 개수를 조정하여 초반 난이도를 조절할 수 있습니다.
*   **`Game.getTotalXPForLevel()`**: 레벨업에 필요한 경험치 공식을 수정하여 레벨업 속도를 조절할 수 있습니다.
*   **아이템 확률/데미지**: `Game.UPGRADES_DATA` 내 각 아이템의 `chances` 및 `damages` 배열 값을 조정하여 아이템의 효율을 조절할 수 있습니다.
    * `chances` 및 `damages` 배열은 `maxLevel` 과동일한 크기여야 합니다.

### 2. 게임 시스템 설명

#### 2.1. 캐릭터 (Player Paddle)

*   **시작**: 게임 시작 시 플레이어는 10 데미지를 가진 공 5개를 가지고 시작합니다.
*   **크기**: 40x40 픽셀의 정사각형 블록.
*   **이동**: 이동 기능이 없고 게임 시작 시 중앙에서 시작하며, 쏘아올린 공 중 `첫번째 공이 바닥으로 떨어진 위치` 에서 다음 턴을 시작합니다.
*   **HP**: 초기 1000 HP를 가지며, 적이 바닥에 닿으면 해당 적의 현재 HP의 1/10만큼 HP가 감소합니다.
*   **HP 표시**: 캐릭터 위에 현재 HP 수치와 HP 바가 표시됩니다. HP 바는 HP 비율에 따라 색상이 변합니다 (녹색 -> 노란색 -> 빨간색).
*   **피격 이펙트**: 적에게 HP가 깎일 때 캐릭터가 잠시 빨갛게 번쩍이며 살짝 흔들립니다.

#### 2.2. 공 (Balls, 메인공)

*   **준비**: 마우스를 움직이면 마우스를 따라 발사 각도를 지정합니다. 공이 발사될 각도의 경로를 화면에서 볼 수 있습니다.
*   **발사**: 마우스 클릭 시 캐릭터 상단 중앙에서 마우스 방향으로 공을 발사합니다.
*   **충돌**: 벽과 적에 부딪히면 튕겨나갑니다. 벽과 적에 충돌 시에는 공의 방향과 속도 벡터를 기반으로 자연스러운 반사 각도를 계산합니다. (matter-js 활용)
*   **소멸**: 화면 하단으로 벗어나면 소멸합니다.
*   **멀티볼 생성 공 (새끼공)**: `멀티볼` 아이템에 의해 생성된 공은 다른 색으로 표시됩니다. `새끼공` 은 추가적인 다른 `아이템` 효과를 발동시키지 않습니다.

#### 2.3. 적 (Enemies)

*   **크기**: 40x40 픽셀의 정사각형 블록.
*   **생성**: 게임 시작 시 3줄이 생성되며, 매 턴 시작 시 새로운 적 줄이 추가됩니다. 화면에 적이 한마리도 없으면 모든 공을 강제 회수합니다.
*   **하강**: 매 턴 시작 시 모든 적이 한 칸씩 아래로 부드럽게 애니메이션됩니다.
*   **종류**: NORMAL, ELITE, UNIQUE, SPECIAL, BOSS 5가지 종류가 있으며, 각각 다른 HP, 크기, 색상, 타격음을 가집니다.
    *   **색상**: NORMAL: `silver`, ELITE: `limegreen`, UNIQUE: `dodgerblue`, SPECIAL: `mediumorchid`, BOSS: `gold`
    *   **크기**: NORMAL: `40x40`, ELITE: `80x40, 40x80`, UNIQUE: `120x40, 40x120`, SPECIAL: `80x80`, BOSS: `160x160`
*   **HP 증가**: 5턴마다 등장하는 적의 기본 HP가 `(현재 턴 / 5) * 50`만큼 증가합니다.
*   **HP 표시**: 각 적 블록 상단 내부에 HP 바와 현재 HP 수치가 표시됩니다.
*   **피격 이펙트**: 공에 맞으면 빨간색으로 잠시 번쩍이며, 공이 튕겨나간 반대 방향으로 밀려났다가 돌아오는 듯한 반동 효과를 가집니다. (matter-js 활용)
*   **겹침 방지**: 적 생성 시 그리드 점유율을 정확히 계산하여 적들이 절대로 겹쳐서 나타나지 않도록 합니다.
*   **스페셜 적 등장 규칙**:
    *   최소 10턴 이후부터 등장 가능합니다.
    *   한 번 등장하면 최소 5턴 동안은 다시 등장하지 않습니다.
    *   조건이 충족되지 않으면 `NORMAL` 적으로 대체됩니다.
*   **보스 적 등장 규칙**:
    *   20턴 마다 한번 씩 등장합니다.
    *   보스 등장 후 보스가 죽을 때 까지는 턴이 진행되더라도 추가 적이 생성되지 않습니다.

#### 2.4. 레벨업 시스템

*   **경험치 획득**: 없앤 적의 `maxHp` 만큼의 경험치(XP)를 획득합니다.
*   **레벨업 조건**: `Game.cumulativeXP`가 현재 레벨의 `Game.getNextLevelTargetXP()`를 초과하면 레벨업합니다.
*   **경험치 테이블**: `Game.getTotalXPForLevel()` 함수에 의해 계산됩니다. 초반 레벨업은 비교적 빠르지만, 레벨이 높아질수록 점진적으로 더 많은 경험치를 요구합니다.
*   **스킬/아이템 선택**: 턴 종료 시 레벨업이 발생하면 게임이 일시 정지되고, 3가지(또는 그 이하)의 랜덤한 스킬/아이템 선택지가 모달로 나타납니다. 플레이어는 이 중 하나를 선택하여 적용합니다.

#### 2.5. 게임 진행 속도 조절

*   **배속 기능**: 마지막 공을 발사한 시점부터 10초가 지나면 5배속, 20초가 지나면 10배속으로 게임 물리 연산이 가속됩니다. 화면 우측 상단에 'SPEED UP x5' 또는 'SPEED UP x10'이 표시됩니다.
*   **'공 부르기' 기능**: 공을 발사한 시점부터 5초가 지나도 턴이 끝나지 않으면 화면 하단에 '공 부르기' 버튼이 나타납니다. 이 버튼을 클릭하면 남아있는 모든 공이 즉시 바닥으로 강제 회수되어 턴을 종료합니다.

### 3. 게임의 룰 설명

1.  **목표**: 화면 상단에서 내려오는 적들을 공으로 공격하여 제거하고, 최대한 높은 점수와 레벨을 달성하세요.
2.  **조작**:
    *   **마우스 이동**: 발사할 공의 각도를 결정합니다.
    *   **마우스 클릭**: 캐릭터 상단 중앙 위치에서 마우스 커서 방향으로 공을 발사합니다.
3.  **턴 진행**:
    *   처음으로 바닥에 내려오는 공의 위치에서 다음 턴을 수행하게 됩니다.
	*   처음으로 바닥에 공이 떨어지면 플레이어가 해당 위치로 부드럽게 애니메이션 되어 어느 위치에서 다음 턴을 시작하게 될지 볼 수 있습니다.
    *   모든 공이 화면 하단으로 사라지면 한 턴이 종료됩니다.
    *   턴이 종료되면 다음 턴이 시작되기 전 새로운 적들이 한 칸 아래로 애니메이션 되며 내려옵니다.
    *   레벨업이 필요한 경우, 턴 종료 시 스킬/아이템 선택 모달이 나타납니다.
4.  **게임 오버**:
    *   적이 화면 하단의 캐릭터 라인까지 내려오면 캐릭터 HP가 감소합니다.
    *   캐릭터 HP가 0이 되면 게임 오버됩니다.
5.  **점수**: 적을 처치하면 점수를 획득합니다. (NORMAL: 100, ELITE: 200, UNIQUE: 500, SPECIAL: 1000, BOSS: 10000)
6.  **레벨업**: 처치한 적에 따라 경험치를 획득하고, 일정 경험치에 도달하면 레벨업하여 새로운 스킬이나 아이템을 선택할 수 있습니다.

### 4. 스킬/아이템 설명

레벨업 시 3가지 랜덤 선택지 중 하나를 선택하여 캐릭터를 강화할 수 있습니다.

#### 4.1. 스킬 (Skill)

*   **공격력 증가**:
    *   **효과**: 공의 기본 데미지가 +1 증가합니다.
    *   **최대 레벨**: 999
*   **공 추가**:
    *   **효과**: 한 턴에 발사할 수 있는 공의 개수가 +1 증가합니다.
    *   **최대 레벨**: 999

#### 4.2. 아이템 (Item)

*   **멀티샷 (Multi Shot)**:
    *   **효과**: 적 타격 시 확률적으로 현재 공 위치에서 랜덤 방향으로 추가 공 2개를 발사합니다.
    *   **최대 레벨**: 3
    *   **레벨별 효과**:
        *   Lv1: 발동 확률 20%, 추가 공 데미지 20%
        *   Lv2: 발동 확률 30%, 추가 공 데미지 25%
        *   Lv3: 발동 확률 40%, 추가 공 데미지 30%
    *   **특수**: 추가 생성된 공은 반투명한 `lightskyblue` 색으로 표시되며, 추가 `아이템` 효과를 발동시키지 않습니다.
    *   **사운드**: 발동 시 `multi.mp3` 재생.

*   **폭발탄 (Explosive Shell)**:
    *   **효과**: 적 타격 시 확률적으로 해당 적을 중심으로 7x7 영역 내 다른 적들에게 스플래쉬 데미지를 입힙니다.
    *   **최대 레벨**: 3
    *   **레벨별 효과**:
        *   Lv1: 발동 확률 20%, 스플래쉬 데미지 50%
        *   Lv2: 발동 확률 30%, 스플래쉬 데미지 75%
        *   Lv3: 발동 확률 40%, 스플래쉬 데미지 100%
    *   **특수**: 멀티샷에 의해 생성된 공은 폭발탄 효과를 발동시키지 않습니다.
    *   **시각 효과**: 발동 시 타격 지점에 반투명한 검은색 원형 충격파가 표시됩니다.
    *   **사운드**: 발동 시 `explo.mp3` 재생.

*   **레이저 (Laser Splitter)**:
    *   **효과**: 적 타격 시 확률적으로 타격된 적의 중앙 Y 좌표와 동일한 선상에 있는 모든 적에게 데미지를 줍니다.
    *   **최대 레벨**: 3
    *   **레벨별 효과**:
        *   Lv1: 발동 확률 20%, 레이저 데미지 20%
        *   Lv2: 발동 확률 30%, 레이저 데미지 25%
        *   Lv3: 발동 확률 40%, 레이저 데미지 30%
    *   **특수**: 멀티샷에 의해 생성된 공은 레이저 효과를 발동시키지 않습니다.
    *   **시각 효과**: 발동 시 타격된 적의 중앙 Y 좌표를 기준으로 화면을 가로지르는 `lightcyan` 색의 레이저 라인이 잠시 표시됩니다.
